    List中RemoveAll方法的注意事项

    遇到了remove方法不产生作用的情况，遂Google寻求解决方法，发现还是自己基础薄弱啊，掉进了一个很难察觉的陷阱

    removeAll API如下

    boolean removeAll(Collection<?> c)  

从列表中移除指定 collection 中包含的其所有元素（可选操作）。

    很多时候我们没有注意到这个API中包含的一个陷阱，在移除的时候，Java会先进行判断包含的操作，当对自定义的对象进行操作时，由于没有Override hashCode和equals方法，就会出现上述问题，所以当使用自定义对象进行Collection操作时，建议首先对hashCode和equals方法进行Override，以免出现不必要的问题
